# macOS Windows Explorer 风格文件管理器开发计划

## 项目定位

我们要做的不是 MVP，也不是一个简单 Demo，而是一个功能完整、UI 好看、可以长期日常使用的 macOS 文件管理器产品。

产品目标：

- 面向从 Windows 转到 macOS、但不喜欢 Finder 交互方式的用户。
- 展示结构和操作习惯尽量接近 Windows Explorer。
- 保留 macOS 原生性能、系统能力和视觉质感。
- 第一眼要像成熟产品，而不是临时工具。
- 文件浏览、文件操作、搜索、预览、标签页、收藏、权限处理、性能优化都要纳入正式范围。

## 技术选型

推荐使用原生 macOS 技术栈：

- 主语言：Swift
- UI 框架：SwiftUI + AppKit
- 文件系统：FileManager、URLResourceValues
- 系统打开方式和文件图标：NSWorkspace
- 文件变化监听：FSEvents
- 文件预览：Quick Look
- 文件拖拽：NSItemProvider、NSPasteboard、AppKit Drag and Drop
- 文件操作进度：Progress、OperationQueue、Swift Concurrency
- 数据持久化：UserDefaults + JSON 配置文件，后期按需要加入 SQLite
- 架构模式：MVVM + Service Layer
- 开发工具：Xcode
- 推荐最低系统：macOS 14，优先适配 macOS 15 及以上

不建议第一版使用 Electron。文件管理器需要高质量系统集成，包括文件权限、右键菜单、拖拽、图标、预览、快捷键、沙盒授权和大目录性能。原生 Swift 路线更适合做成长期产品。

## UI 设计目标

UI 必须作为核心产品能力开发，不放到最后补救。

视觉关键词：

- 美观
- 干净
- 稳定
- 专业
- 信息密度高
- 接近 Windows Explorer 的熟悉感
- 符合 macOS 的细腻质感

设计原则：

- 首屏就是完整文件管理器，不做介绍页。
- 工具栏、地址栏、侧边栏、文件列表必须层级清楚。
- 不使用花哨大渐变、装饰性背景、营销式卡片。
- 文件列表是主角，视觉不能喧宾夺主。
- 按钮优先使用图标，必要时配合文字。
- 不用大圆角堆卡片，整体要像生产力工具。
- 所有按钮、菜单、行项目都要有 hover、pressed、selected、disabled 状态。
- 适配浅色模式和深色模式。
- 小窗口下不允许文字重叠、工具栏挤压错位。

### 主界面布局

主窗口由五个区域组成：

- 顶部工具栏：返回、前进、上一级、刷新、视图切换、新建、搜索。
- 地址栏：可在面包屑模式和文本输入模式之间切换。
- 左侧边栏：快速访问、收藏夹、系统位置、磁盘。
- 中央文件区：列表视图、图标视图、搜索结果视图。
- 底部状态栏：项目数量、选中数量、选中大小、后台任务状态。

可选区域：

- 右侧详情面板。
- 右侧预览面板。
- 底部文件操作进度抽屉。

### UI 质量验收标准

- 应用启动后第一屏必须像一个完整产品。
- 文件列表行高、列宽、图标大小稳定，不因文字变化导致布局跳动。
- 长文件名要截断或换行处理，不能覆盖其他列。
- 地址栏在编辑和面包屑模式切换时不能明显抖动。
- 深色模式下对比度充足。
- 选中态清楚，焦点态清楚。
- 空目录、无权限、加载中、搜索无结果都有专门状态。
- 所有主要操作都能通过菜单、右键或快捷键找到。

### 视觉规格

整体尺寸建议：

- 主工具栏高度：44 到 52 pt。
- 地址栏高度：30 到 34 pt。
- 侧边栏宽度：220 到 280 pt，可拖拽调整。
- 文件列表行高：28 到 34 pt。
- 图标视图单元格：96 到 128 pt。
- 状态栏高度：24 到 28 pt。

颜色方向：

- 浅色模式以系统背景色为基础，文件区保持明亮干净。
- 深色模式以系统深色背景为基础，避免纯黑和过重边框。
- 选中态使用系统强调色，但要降低大面积刺眼感。
- 分割线要轻，主要靠间距、背景层级和 hover 状态建立结构。

组件要求：

- 工具栏按钮使用统一尺寸的图标按钮。
- 地址栏既要像 Windows Explorer 一样可编辑，也要符合 macOS 的输入质感。
- 文件列表列头支持点击排序，排序列和方向必须可见。
- 右键菜单使用系统菜单样式，菜单项按常用频率排序。
- 文件操作进度使用非阻塞浮层或底部抽屉，不打断浏览。
- 错误提示要具体说明原因和可执行动作。

## 架构设计

建议目录结构：

```text
MacExplorer/
  App/
    MacExplorerApp.swift
    AppCommands.swift
  Features/
    Browser/
    Sidebar/
    Toolbar/
    AddressBar/
    FileList/
    Preview/
    Search/
    Tabs/
    Preferences/
    FileOperations/
  Core/
    Models/
    Services/
    Stores/
    Utilities/
  DesignSystem/
    Components/
    Icons/
    Theme/
  Resources/
  Tests/
```

核心模块：

- `FileSystemService`：读取目录、获取文件属性、监听变化。
- `FileOperationService`：复制、移动、删除、重命名、冲突处理。
- `NavigationStore`：当前路径、返回前进历史。
- `TabStore`：标签页状态。
- `SidebarStore`：收藏夹、系统位置、磁盘列表。
- `SearchService`：异步搜索和取消。
- `PreviewService`：Quick Look 和基础预览。
- `PreferencesStore`：用户偏好和目录视图偏好。
- `IconCache`：文件图标缓存。
- `MetadataCache`：文件属性缓存。

## 功能完整性验收矩阵

开发过程中按以下矩阵判断产品是否完整：

| 模块 | 必须完成的能力 | 不可接受的状态 |
| --- | --- | --- |
| 浏览 | 列表视图、图标视图、排序、刷新、隐藏文件、大目录加载 | 只能打开固定目录，或大目录卡死 |
| 导航 | 返回、前进、上一级、地址栏、面包屑、侧边栏 | 只能靠双击目录进入，不能直接跳路径 |
| 文件操作 | 新建、复制、剪切、粘贴、重命名、删除、拖拽、冲突处理 | 任何高频操作需要回到 Finder |
| 搜索 | 当前目录、递归、过滤、取消、结果路径 | 搜索阻塞 UI，或只能搜索当前可见列表 |
| 预览 | Quick Look、详情面板、基础缩略图 | 选中文件后没有任何上下文信息 |
| 标签页 | 新建、关闭、恢复、独立历史 | 只能单窗口单路径工作 |
| 偏好 | 视图、排序、隐藏文件、侧边栏、窗口状态 | 每次打开都恢复默认状态 |
| 权限 | 授权引导、安全书签、错误提示 | 遇到无权限目录直接失败或静默失败 |
| 性能 | 异步加载、图标缓存、元数据缓存、变化监听 | UI 经常冻结或文件变化不刷新 |
| UI | 浅色深色、状态完整、布局稳定、视觉统一 | 看起来像临时 Demo 或控件堆砌 |
